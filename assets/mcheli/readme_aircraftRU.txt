★★★★★ Функция загрузки воздушных судов

; Функция загрузки воздушных судов работает, когда:
; Хозяин (хост) указывает воздушное судно, которое он может загрузить (AddRack).
; Воздушное судно указывает номер стойки (RideRack).

AddRack = контейнер, 0.0, 1.4, -4.7, 0.0, 1.0, -16.1
AddRack = контейнер / ah-64, 0.0, 1.4, -4.7, 0.0, 1.0, -16.1, 5.0, 20
AddRack = вертолет/транспортное средство / t-4, 0.0, 1.4, -4.7, 0.0, 1.0, -16.1, 5.0, 100000, 0.0, 0.0
; Это настройка для носителя.
; Параметр 1: Название сущности, которую можно загрузить.
; Параметры 2~4: Координаты стойки.
; Параметры 5~7: Координаты входа/выхода X, Y, Z. Сущность может быть загружена, когда она находится рядом с этими координатами. Также она выгружается оттуда.
; Пример: для AC-130 установите координаты стойки внутри воздушного судна, а вход/выход рядом с задним люком.
; Параметр 8: Радиус (дистанция) от входа. Эти параметры и последующие — необязательные.
; Параметр 9: Высота, на которой парашют раскрывается. Установка очень высокого значения предотвратит раскрытие парашюта.
; Параметр 10: Горизонтальный угол загруженной сущности.
; Параметр 11: Вертикальный угол загруженной сущности.

Стойка, позволяющая загрузить контейнеры, вертолеты и другие сущности на воздушное судно, как мобы.
Сущности могут быть указаны по имени с использованием комбинации:
;контейнер → Все контейнеры
;вертолет → Все вертолеты
;самолет → Все самолеты с фиксированными крыльями
;транспортное средство → Все наземные транспортные средства
;Конкретное название воздушного судна → например, ah-64, t-4, s-75

RideRack = c5, 1

; RideRack = [Название воздушного судна], [Номер стойки (1~)]
; Используется, когда сущность должна быть на другом воздушном судне.

; Пример: ExclusionSeat = 3, 4, 5
; Если моб находится на сиденье 3, сиденья 4 и 5 не могут быть использованы.
; Если моб находится на сиденье 4, сиденья 3 и 5 не могут быть использованы.
; Если моб находится на сиденье 5, сиденья 3 и 4 не могут быть использованы.
; Номера сидений/стойк назначаются последовательно в порядке появления в конфигурационном файле:
; AddSeat → Сиденье 1
; AddGunnerSeat → Сиденье 2
; AddSeat → Сиденье 3
; AddRack → Стойка 4
; AddRack → Стойка 5
; AddRack → Стойка 6
; Рекомендуется сначала перечислить все сиденья, затем стойки для ясности.

; Слишком маленькое значение приведет к плохому соответствию транспортного средства местности и может вызвать застревание в блоках

CameraRotationSpeed = 25
; Скорость поворота камеры
; Для танков это можно использовать для ограничения скорости поворота башни

WeightType = Tank
; Тип веса транспортного средства: Танковый, Автомобильный или Неизвестный
; Tank : Не повреждается при столкновении с мобами, разрушает больше блоков
; Car  : Получает урон при столкновении с мобами, разрушает меньше блоков
; Поведение для типов, отличных от Tank или Car, не определено
; Количество разрушенных блоков можно настроить в файле mcheli.cfg

WeightedCenterZ = 0.0
; Настройка координаты центра масс по оси Z
; Координата центра масс, когда транспортное средство наклоняется, чтобы соответствовать местности
; * Эта настройка может не работать корректно, поэтому если ощущается дискомфорт, лучше не использовать ее

SetWheelPos = 1.75, -0.24, 4.85, 3.02, 1.44, -1.54, -2.91
; SetWheelPos = координата X, координата Y, координата Z1, Z2, Z3 ...
; Указывает точку контакта с землей. Транспортное средство будет наклоняться в зависимости от этой точки.
; Нет необходимости устанавливать отрицательное значение для координаты X
; ★ Рекомендуется установить координату Y равной -0.24, независимо от типа транспортного средства
